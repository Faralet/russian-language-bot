# Changelog

## v3.3 - контент 500 + интервальное повторение

- Контент: +222 упражнения (итого ровно 500) в новом файле
  `app/data/seed_content_extra.py`; проверено программно: ни одного дубля
  с существующей базой, у каждого ровно один правильный ответ, правильный
  вариант стоит первым лишь в ~49% случаев. Догружается в существующую
  базу автоматически при старте.
- Интервальное повторение (1-3-7 дней): таблица `user_exercise_reviews`
  (создается автоматически), `update_review_schedule()` в момент ответа,
  просроченные повторы идут первыми в «Занятии дня» (до 2 из 5) и в
  «Моих ошибках»; после прохождения всех интервалов запись удаляется.
  Любая новая ошибка возвращает правило в начало цикла.
- «Мои ошибки»: вместо отката к случайной тренировке при нехватке
  материала - добор обычными вопросами (повторы всегда первые).
- После неверного ответа бот сообщает, что вернется с вопросом завтра.
- DEPLOY_GUIDE.md - инструкция развертывания «для школьника».
- Тесты: 47 unit, 49 e2e (полный цикл повторения 1-3-7 на реальном PostgreSQL).

- Велком-бонус: `effective_daily_limit()` удваивает дневной лимит первые
  `WELCOME_BONUS_DAYS` (3) дня после регистрации; упоминание в приветствии.
- Ежедневная сводка владельцу: `build_admin_report()` + `send_admin_reports()`
  в общем планировщике; отчет за вчера в таймзоне приложения (новые/активные

  по дню; отправка только админам, которые уже писали боту.
  Настройки: `ENABLE_ADMIN_REPORT=true`, `ADMIN_REPORT_TIME=09:00`.
- Исправление: конфликт уникального индекса при двойном клике мог всплыть
  на autoflush (внутри upsert_topic_progress), а не на commit - try/except
  теперь охватывает всю запись ответа.
- Тесты: 44 unit, 43 e2e на реальном PostgreSQL (велком-бонус, сводка,
  дедупликация сводки).

## v3.1 - retention-пакет

# Changelog v3 (после аудита v2)

## Подтверждено аудитом (без изменений)

- Парсинг `ADMIN_TELEGRAM_IDS` надежен: «123», «123,456», пустое значение, пробелы.
- `callback.from_user` используется корректно во всех callback-сценариях.
- Проверка принадлежности занятия пользователю и статуса `started` - на месте.
- Seed-контент: ровно 278 упражнений, разбивка совпадает с заявленной
  (управление 38, паронимы 40, ударения 50, пунктуация 30, орфография 30,
  речевая точность 24, синтаксис 10, ЕГЭ/ОГЭ 56). Дублей нет.
- Платежи и AI выключены по умолчанию.

## Исправления

1. **Дневной лимит по часовому поясу** (`app/services/lesson_service.py`)
   - Что было: «сегодня» считалось по UTC - лимит сбрасывался в 03:00 МСК.
   - Почему ошибка: бриф требует расчет по `APP_TIMEZONE`.
   - Что изменено: `today_bounds_utc()` строит границы суток в таймзоне
     приложения и переводит их в UTC для сравнения с `answered_at`.
   - Как проверить: `pytest tests/test_limits_and_premium.py`.

2. **Гонка при двойном клике** (`app/db/models.py`, `app/db/init_db.py`,
   `app/services/lesson_service.py`)
   - Что было: защита select-then-insert без ограничения в БД; два быстрых
     клика могли пройти проверку одновременно и задвоить статистику.
   - Что изменено: уникальный индекс `(user_id, lesson_id, exercise_id)` на
     `user_answers` + обработка `IntegrityError` с откатом и возвратом уже
     засчитанного результата. Для существующих баз индекс догружается
     безопасной мини-миграцией при старте.

3. **Физическое удаление упражнений** (`app/services/admin_service.py`)
   - Что было: `/delete_exercise` удалял запись; каскад стирал варианты
     ответа и исторические ответы пользователей - ломалась статистика.
   - Что изменено: мягкое удаление `status=deleted`; история сохраняется.

4. **Премиум без даты окончания** - `is_premium=True` с `premium_until=NULL`
   (ручная выдача) теперь считается активным. Проверка премиума
   унифицирована в `is_active_premium()` (раньше дублировалась в 3 местах).

5. **`callback.message` без проверки** - все callback-обработчики получили
   защиту от `None`/недоступного сообщения (хелпер `callback_message`).

6. **Тренировка ошибок недобирала вопросы** - `LIMIT` применялся до
   дедупликации повторных ошибок. Теперь сначала выбираются уникальные
   упражнения по последней ошибке, потом ограничивается количество.

7. **Дубли в сохраненных правилах** - повторное нажатие «Сохранить правило»
   больше не создает дубликат.

8. **`engine.dispose()` во втором event loop** (`app/main.py`) - закрытие
   пула перенесено внутрь основного цикла.
