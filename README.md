# Telegram-бот «Точный русский» / MVP v3.3

Интерактивный Telegram-бот для обучения тонким правилам русского языка: короткие занятия, вопросы, объяснения, прогресс, тренировка ошибок и Telegram-админка для управления контентом.

## Что нового в v3.3 (контент 500 + интервальное повторение)

- **База расширена до 500 упражнений** (+222): ударения 85, паронимы 75, управление 68, пунктуация 55, орфография 60, речевая точность 49, синтаксис 25, ЕГЭ/ОГЭ 83. Все нормы - по орфоэпическому минимуму ФИПИ и академическим словарям.
- **Интервальное повторение ошибок (1-3-7 дней)**: ошибся - вопрос вернется через день, потом через три, потом через семь. Прошел все три интервала - правило считается усвоенным. Повторы автоматически вплетаются в «Занятие дня» (до 2 вопросов) и открывают «Мои ошибки».
- «Мои ошибки» при нехватке материала добирается обычными вопросами, а не подменяется случайной тренировкой.
- **DEPLOY_GUIDE.md**: пошаговая инструкция развертывания для новичка.

## Что нового в v3.2 (блок NOW завершен)

- **Велком-бонус**: первые 3 дня после регистрации дневной лимит удвоен (10 вопросов вместо 5). Настройка: `WELCOME_BONUS_DAYS`.
- **Ежедневная сводка владельцу**: каждое утро администраторы получают в Telegram отчет за вчера - новые и активные пользователи, ответы, точность, завершенные занятия, самая сложная тема. Настройки: `ENABLE_ADMIN_REPORT`, `ADMIN_REPORT_TIME` (по умолчанию 09:00).
- Усилена обработка гонки при двойном клике (перехват конфликта на autoflush).

## Что нового в v3.1 (retention-пакет)

- **Ежедневные напоминания**: одно сообщение в день в выбранное время с «языковой деталью дня» и кнопкой запуска занятия. Настройка: ⚙️ Настройки → 🔔 Напоминания (вкл/выкл, время). Если пользователь заблокировал бота, напоминания отключаются автоматически.
- **Умный подбор вопросов**: упражнения, на которые пользователь уже отвечал, не повторяются, пока есть новые; добор отвечанными - только когда новые закончились.
- **Персонализация по цели**: при цели «Готовлюсь к ЕГЭ/ОГЭ» около 40% вопросов занятия берутся из экзаменационной темы.
- **Кнопки А/Б/В**: если вариант ответа длинный, варианты показываются в тексте вопроса, а на кнопках - буквы (Telegram обрезает длинный текст кнопок).
- **Финал занятия**: результат, серия дней (streak) и кнопки «Еще занятие», «Повторить ошибки», «Напоминать каждый день» - без тупика в конце.

## Что нового в v3

- Дневной лимит теперь сбрасывается в полночь по `APP_TIMEZONE` (раньше - по UTC).
- Защита от двойного клика усилена уникальным индексом в БД: ответ физически не может засчитаться дважды.
- `/delete_exercise` стал безопасным: мягкое удаление (`status=deleted`), история ответов и статистика сохраняются.
- Новые админ-команды: `/exercise ID` (просмотр), `/search_exercises текст` (поиск), `/export_exercises [статус]` (выгрузка в JSON-файл).
- Раздел «📌 Сохраненные правила» в настройках: сохраненные правила теперь можно посмотреть.
- Прогресс расширен: занятия, серия дней (streak), сильные и слабые темы.
- После ответа клавиатура вариантов снимается с вопроса.
- Premium, выданный вручную без даты окончания, считается активным.
- Строгая валидация JSON-упражнений: `level`, `type`, `status`, пустые строки, дубли вариантов.
- Добавлены unit-тесты (`tests/`, запуск: `pip install -r requirements-dev.txt && pytest`).
- Порт PostgreSQL больше не открыт наружу (привязан к 127.0.0.1).

## Быстрый запуск на VPS

```bash
cp .env.example .env
nano .env
```

Заполните:

```env
BOT_TOKEN=токен_от_BotFather
ADMIN_TELEGRAM_IDS=ваш_Telegram_ID
ENABLE_PAYMENTS=false
```

Запуск:

```bash
docker compose up --build -d
```

Логи:

```bash
docker compose logs -f bot
```

## Основные команды

- `/start` - запуск бота.
- `/admin` - админ-панель.
- `/add_exercise` - добавить упражнение или массив упражнений JSON.
- `/edit_exercise ID` - исправить упражнение JSON-патчем.
- `/exercise ID` - посмотреть упражнение, варианты и статистику.
- `/search_exercises текст` - найти упражнения по вопросу, объяснению, теме или тегам.
- `/export_exercises [статус]` - выгрузить упражнения в JSON-файл.
- `/publish_exercise ID` - опубликовать упражнение.
- `/archive_exercise ID` - убрать упражнение из выдачи.
- `/delete_exercise ID` - мягко удалить упражнение (история ответов сохраняется).

## Как добавить упражнение

Откройте `/admin` → `➕ Добавить JSON` и отправьте JSON:

```json
{
  "topic_slug": "governing",
  "level": "basic",
  "question": "Как правильно?",
  "options": [
    {"text": "Согласно приказу", "is_correct": true},
    {"text": "Согласно приказа", "is_correct": false}
  ],
  "short_explanation": "После «согласно» нужен дательный падеж: согласно чему? приказу.",
  "full_explanation": "Слово «согласно» требует дательного падежа: согласно приказу, договору, расписанию.",
  "example_text": "Согласно расписанию, встреча начнется в 10:00.",
  "interesting_fact": "Ошибка часто встречается в деловой речи.",
  "tags": ["управление", "падежи"],
  "difficulty_score": 2,
  "status": "published"
}
```

Можно отправить массив таких объектов или `.json` файл.

## Как исправить упражнение

```text
/edit_exercise 125
```

Потом отправьте JSON-патч:

```json
{
  "short_explanation": "Исправленное объяснение.",
  "status": "published"
}
```

## Обновление без удаления базы

Если бот уже стоит на VPS:

```bash

docker compose up --build -d
```

Новые упражнения из seed-контента будут добавлены автоматически, старые пользовательские данные сохранятся.

Тесты покрывают: парсинг `ADMIN_TELEGRAM_IDS`, валидацию JSON-упражнений,
границы дневного лимита по часовому поясу, логику премиума и целостность
seed-контента (500 упражнений, у каждого ровно один правильный ответ).

## Как узнать свой Telegram ID

Напишите боту `@userinfobot` в Telegram - он пришлет ваш числовой ID.
Этот ID нужно указать в `.env` в переменной `ADMIN_TELEGRAM_IDS`.
