# Как разместить бота в Telegram: пошаговая инструкция

Эта инструкция написана для человека, который никогда не разворачивал ботов.
Делайте шаг за шагом, ничего не пропуская. Время на все: 30-40 минут.

**Что получится в конце:** ваш бот живет на арендованном сервере 24/7,
и **любой человек в Telegram** может найти его по имени, нажать «Старт»
и заниматься. Пользователям ничего устанавливать не нужно. Лимиты,
премиум-витрина и админка работают автоматически.

---

## Шаг 1. Создаем бота у BotFather (5 минут)

1. Откройте Telegram и в поиске наберите `@BotFather` (синяя галочка).
2. Нажмите «Старт» и отправьте команду `/newbot`.
3. BotFather спросит **имя** - это то, что видят люди. Например: `Точный русский`.
4. Потом спросит **username** - он должен быть уникальным и заканчиваться
   на `bot`. Например: `tochny_russkiy_bot`.
5. В ответ придет сообщение с **токеном** - длинная строка вида
   `1234567890:AAEhBOweik6ad9r_QXMENQT-uGpW8mzz1y4`.

> ⚠️ **Токен - это ключ от бота.** Никому его не показывайте и не
> отправляйте в чаты. Кто знает токен - тот управляет ботом.

Сохраните токен в заметки - он понадобится на шаге 6.

## Шаг 2. Узнаем свой Telegram ID (1 минута)

1. В поиске Telegram наберите `@userinfobot` и нажмите «Старт».
2. Бот пришлет ваш `Id` - число вида `123456789`.

Сохраните его рядом с токеном. Это число сделает вас администратором.

## Шаг 3. Арендуем сервер (10 минут)

Подойдет любой VPS-хостинг (Timeweb Cloud, Beget, reg.ru, Selectel и др.).

При заказе выбирайте:

- Операционная система: **Ubuntu 22.04** или **24.04**
- Ресурсы: **1 CPU, 2 ГБ RAM, 20 ГБ диска** (самый дешевый тариф, ~300-500 ₽/мес)
- Домен **не нужен**, дополнительные услуги не нужны

После оплаты хостинг пришлет на почту:

- **IP-адрес** сервера (например, `185.42.13.7`)
- логин **root** и **пароль**

## Шаг 4. Подключаемся к серверу (5 минут)

На вашем компьютере с Windows:

1. Нажмите `Win + R`, введите `powershell`, нажмите Enter.
2. В синем окне введите (подставьте свой IP):

```powershell
ssh root@185.42.13.7
```

3. На вопрос `Are you sure you want to continue connecting?` введите `yes`.
4. Введите пароль из письма хостинга. **Пароль при вводе не отображается** -
   это нормально, просто введите и нажмите Enter.

Если приветствие сменилось на `root@...:~#` - вы внутри сервера.

## Шаг 5. Устанавливаем Docker (5 минут)

Скопируйте и вставьте в окно сервера (вставка в PowerShell - правая кнопка мыши):

```bash
apt update && apt install -y unzip curl
curl -fsSL https://get.docker.com | sh
systemctl enable docker
```

Дождитесь окончания (1-3 минуты). Проверка:

```bash
docker --version
```

Должно ответить что-то вроде `Docker version 27...`.

## Шаг 6. Загружаем бота на сервер (5 минут)

Архив `russian_language_bot_v3.3_content500.zip` лежит в вашей папке проекта.

**Откройте НОВОЕ окно PowerShell на своем компьютере** (не закрывая окно
с сервером) и выполните (подставьте свой путь и IP):

```powershell
scp "C:\Users\maxry\Desktop\Project\ТГ\russian_language_bot\TG Rus BOT\russian_language_bot_v3.3_content500.zip" root@185.42.13.7:/root/
```

Введите пароль сервера. Когда файл скопируется, **вернитесь в окно сервера**:

```bash
cd /root
unzip russian_language_bot_v3.3_content500.zip
cd russian_language_bot
```

## Шаг 7. Заполняем настройки (5 минут)

```bash
cp .env.example .env
nano .env
```

Откроется текстовый редактор. Стрелками дойдите до строк и впишите свои
значения (из шагов 1 и 2):

```env
BOT_TOKEN=1234567890:AAEhBOweik6ad9r_QXMENQT-uGpW8mzz1y4
ADMIN_TELEGRAM_IDS=123456789
```

Если админов двое - через запятую: `ADMIN_TELEGRAM_IDS=123456789,987654321`

Остальные строки не трогайте. Сохранение в nano:

1. `Ctrl + O`, затем `Enter` - сохранить
2. `Ctrl + X` - выйти

## Шаг 8. Запускаем (3 минуты)

```bash
docker compose up --build -d
```

Первый запуск собирает все 2-4 минуты. Затем проверьте:

```bash
docker compose ps
```

Обе строки (`db` и `bot`) должны быть в состоянии `Up` / `running`.
Посмотреть, что бот пишет в логи:

```bash
docker compose logs -f bot
```

(выход из логов - `Ctrl + C`; сам бот при этом продолжает работать)

## Шаг 9. Проверяем в Telegram (2 минуты)

1. Найдите своего бота по username из шага 1.
2. Нажмите «Старт» - придет приветствие и меню.
3. Нажмите «📚 Занятие дня» - пройдите пару вопросов.
4. Отправьте `/admin` - откроется админ-панель (она видна только вам).

**Готово. Бот работает.** Можно отправлять ссылку `t.me/ваш_username_bot`
кому угодно - бот публичный, регистрация не нужна, лимиты для каждого
пользователя бот считает сам.

## Шаг 10. Наводим красоту (по желанию, 5 минут)

Вернитесь к `@BotFather`:

- `/setdescription` - описание, которое видно до нажатия «Старт»
- `/setabouttext` - короткий текст в профиле бота
- `/setuserpic` - аватар
- `/setcommands` - подсказки команд. Отправьте боту BotFather такой список:

```text
start - Запустить бота и открыть меню
```

(админ-команды добавлять в список не нужно - они служебные)

## Как обновлять бота

Когда появится новая версия (новый zip):

```bash
cd /root
# загрузите новый zip через scp (как в шаге 6), затем:
cd russian_language_bot
docker compose down          # ВАЖНО: без флага -v, иначе сотрется база!
cd /root
unzip -o новый_архив.zip     # -o перезаписывает старые файлы
cd russian_language_bot
docker compose up --build -d
```

Пользователи, их прогресс и добавленные вручную упражнения сохранятся.
Новые упражнения из обновления добавятся в базу автоматически.

## Как сделать резервную копию базы

Раз в неделю-две выполняйте:

```bash
cd /root/russian_language_bot
docker compose exec db pg_dump -U russian_bot russian_bot > backup_$(date +%F).sql
```

Файл `backup_2026-06-12.sql` можно скачать к себе:

```powershell
scp root@185.42.13.7:/root/russian_language_bot/backup_2026-06-12.sql C:\Users\maxry\Desktop\
```

## Как добавлять упражнения (без программиста)

Прямо в Telegram: `/admin` → «➕ Добавить JSON». Шаблон бот покажет сам.
Подробности - в README.md и scripts/admin_content_guide.md.

Чтобы дать доступ контент-менеджеру: впишите его Telegram ID в
`ADMIN_TELEGRAM_IDS` через запятую (шаг 7) и выполните
`docker compose restart bot`.

## Если что-то пошло не так

| Симптом | Причина и решение |
|---|---|
| В логах `Unauthorized` | Неверный BOT_TOKEN в `.env`. Проверьте, перезапустите: `docker compose restart bot` |
| Бот молчит в Telegram | `docker compose ps` - оба контейнера `Up`? Если нет: `docker compose logs bot` покажет ошибку |
| `/admin` пишет «закрыт» | В `ADMIN_TELEGRAM_IDS` не ваш ID. Проверьте через @userinfobot |
| Сервер перезагрузился | Ничего делать не нужно: `restart: unless-stopped` поднимет бота сам |
| Хочу все стереть и начать заново | `docker compose down -v` (удалит базу!) и `docker compose up --build -d` |

Полезные команды на каждый день:

```bash
docker compose ps              # статус
docker compose logs -f bot     # живые логи
docker compose restart bot     # перезапуск бота
```
